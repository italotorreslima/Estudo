*docker image pull mysql:5.6 - baixa a imagem do mysql na versão determinada
*docker container run myslq:5.6 - rodar o container (nao vai funcionar pq precisa das variaveis de ambiente)
** -e significa que posteriormente viram as variaveis de ambiente para o docker
*docker run -e MYSQL_ROOT_PASSWORD=root --name meu-mysql -p 3306:3306 mysql:5.6 - para rodar o container e definir uma senha que no caso será root e também o nome do bd e tambem refletir o localhost no container do host
*docker run -e MYSQL_ALLOW_EMPTY_PASSWORD=yes --name meu-mysql -p 3306:3306 mysql:5.6 - define sem senha
*docker images - para ver a listagem das imagens
*docker ps -a - lista todos os containers que estao rodando
*docker stop {nome ou id do container} - ele finaliza o container
*docker rm {nome ou id do container} - para remover um container
*docker inspect {nome ou id do container} - lista parametros e propriedades do container
*docker exec -it {id do container} bash - para começar a interagir com o banco
*docker rmi - remove imagens pelo id
*docker start - reinicia um container que havia sido exited
*docker-compose up --build  - builda o projeto e sobe ele pro localhost
